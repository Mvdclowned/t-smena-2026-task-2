Консольное приложение для ведения личной библиотеки. Позволяет добавлять книги, искать их по ключевым словам, отмечать прочитанное и избранное, а также хранить всё в файле `lib.json`.

**О проекте**
- Формат: консольное приложение
- Язык: Python 3
- Интерфейс: таблицы и подсветка через `rich`
- Хранение данных: JSON-файл `lib.json` в корне проекта

**Базовый функционал**
- Добавление книги: название, автор, жанр, год, описание
- Просмотр списка книг
- Сортировка по названию, автору и году
- Фильтрация по жанру и статусу (прочитана или нет)
- Избранное: добавление, удаление и просмотр списка
- Изменение статуса прочитано/не прочитано
- Поиск по названию, автору и описанию
- Удаление книги
- Сохранение данных в `lib.json`
- Загрузка данных из `lib.json` при старте

**Дополнительно реализовано**
- Пагинация списка по 10 книг на страницу
- Оценки 0–5 и отображение звездами
- Сортировка по оценке
- Редактирование карточки книги
- Статистика по библиотеке
- Импорт книг из внешнего JSON
- Обновление базы из файла без перезапуска приложения
- Улучшенный вывод таблиц и подсветка через `rich`

**Установка и запуск**
1. Установить зависимость:

```bash
pip install rich
```

2. Запустить приложение:

```bash
python main.py
```

**Меню**
- `1` — добавить книгу
- `2` — все книги (сортировка, фильтрация, пагинация)
- `3` — избранные
- `4` — статусы (прочитано/не прочитано, избранное, оценка)
- `5` — редактирование книги
- `6` — удаление
- `7` — поиск
- `8` — статистика
- `9` — импорт из JSON
- `10` — обновить базу из файла
- `0` — выход

**Формат данных**
Хранится список словарей со следующими ключами:
- `id` — числовой идентификатор
- `t` — название
- `a` — автор
- `g` — жанр
- `y` — год
- `ds` — описание
- `rd` — прочитана ли книга (`true/false`)
- `fv` — в избранном (`true/false`)
- `rt` — оценка 0–5

**Импорт**
Меню `9` ожидает JSON-файл со списком книг такого же формата. Если каких-то полей нет, будут применены значения по умолчанию. Повторы пропускаются при совпадении `Название + Автор` без учета регистра.

**Структура проекта**
- `main.py` — логика меню и действий пользователя
- `db.py` — загрузка/сохранение данных в `lib.json`
- `ui.py` — вывод таблиц и статистики через `rich`
- `lib.json` — файл базы данных (создается автоматически)

**Примечания**
- Валидация года ограничена вводом строки; корректность формата остается на пользователе.
- Сортировка по году выполняется по строке, поэтому лучше вводить год в формате `YYYY`.
